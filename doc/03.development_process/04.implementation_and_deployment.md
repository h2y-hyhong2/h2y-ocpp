# 💻 04. 구현 및 배포 정의서 (Implementation & Deployment)
> 가상 스레드 극대화와 로컬 맵 및 무인 인스톨러 배포

---

## 1. 개발 런타임 및 가상 스레드 스택
대용량 I/O 대기를 효율적으로 차단하고 CPU[^CPU] 효율성을 높이기 위한 백엔드 코어 스택 구현 요건입니다.

* **Java 25 가상 스레드(Virtual Threads) 연동:**
  * 기존 Spring의 OS 스레드 맵핑(1:1 Thread Pool) 방식 대신 JVM[^JVM] 레벨의 가상 스레드(Carrier Thread에 스케줄링)를 활성화.
  * `application.properties` 내 `spring.threads.virtual.enabled=true` 속성 설정.
  * DB[^DB] 트랜잭션 대기, HTTP 외부 로밍 API[^API] 통신, 파일 입출력 등 대기 시간이 발생하는 I/O 블로킹 구간에서 물리 스레드가 블로킹되지 않고 즉시 다른 가상 스레드 작업을 수행하도록 하여 1대 VM에서도 동시 2,000개 이상의 스레드를 가볍게 제어.

## 2. 경량화 튜닝 및 리소스 제어 개발
Phase 1 단일 호스트 가상 머신(VM) 환경에서 미들웨어(Kafka, ClickHouse)가 과도하게 시스템 메모리를 점유해 OOM[^OOM](Out of Memory)으로 사망하는 상황을 예방하기 위한 튜닝 스크립트 설계입니다.

* **Kafka Heap Size 및 Retention 제한:**
  * 환경변수 `export KAFKA_HEAP_OPTS="-Xms512M -Xmx512M"` 설정을 통해 JVM 힙 메모리를 최소화.
  * `server.properties` 내 `log.retention.hours=24` 및 `log.segment.bytes=536870912`(512MB) 설정으로 하루가 지난 백로그 세그먼트는 자동 삭제하여 디스크 고갈 위협 차단.
* **ClickHouse 메모리 쿼리 임계치 제어:**
  * `users.xml` 파일에 `<max_memory_usage>4000000000</max_memory_usage>` 설정 추가. 개별 분석 쿼리가 4GB 이상의 메모리를 무단 점유하는 것을 강제 차단하여 전체 OS 런타임 안정성 보장.

## 4. 무인 오프라인 인스톨러 패키징 개발
인프라 구성 요소(MySQL 9.71, ClickHouse, Kafka, Java JVM, App Engine)가 설치되지 않은 호스트 환경에서도 터미널 명령 한 줄(원클릭)로 무인 설치가 완료되도록 빌드 파이프라인을 구축합니다.

* **Linux 환경 (Makeself 아카이브 빌드):**
  * Makeself 유틸리티를 활용하여 각 미들웨어 바이너리 패키지(MySQL 9.71, ClickHouse, Apache Kafka) 및 Spring Boot 어플리케이션 배포 파일(JAR), 그리고 서비스 등록 스크립트를 단일 자동 압축 해제 바이너리 `install.run`으로 컴파일합니다.
  * `install.run` 기동 시, 지정 디렉토리(`/opt/ocpp-lite/`) 아래에 바이너리 아카이브를 추출하고 전용 OS 실행 계정(`ocpp`)을 생성하여 소유권을 할당합니다.
  * **systemd 서비스 등록 및 리소스 제어(Cgroups)**:
    - 각 어플리케이션 및 미들웨어 데몬을 Linux systemd 서비스 유닛(`.service` 파일)으로 등록하여 자동 재기동 및 부팅 시 자동 시작을 설정합니다.
    - 특히 프로세스 격리 부재로 인한 자원 고갈을 방지하기 위해 systemd 서비스 설정 내에서 아래와 같이 리소스 제한(Cgroups)을 강제합니다:
      ```ini
      [Service]
      ExecStart=/usr/bin/java -jar /opt/ocpp-lite/ocpp-core-engine.jar
      MemoryMax=4G
      CPUQuota=50%
      Restart=always
      ```
* **Windows 환경 (Inno Setup 컴파일러):**
  * Inno Setup 컴파일러를 통해 설치용 UI[^UI] 프레임과 미들웨어/애플리케이션 압축본을 단일 실행 파일 `setup.exe`로 패키징합니다.
  * 호스트 환경에 각 바이너리를 해제하고 실행 경로 환경 변수를 자동 셋업합니다.
  * **Windows Service 등록 및 자원 관리**:
    - NSSM(Non-Sucking Service Manager) 유틸리티를 배포판에 포함하여 Spring Boot JAR와 미들웨어 데몬들을 백그라운드 **Windows 서비스(Windows Service)**로 등록합니다.
    - Windows `Job Object` 구성을 통해 서비스들의 프로세스 그룹이 물리 코어 및 힙 메모리 임계치(예: 4GB)를 무단으로 침범할 수 없도록 강제 격리 정책을 셋업 스크립트로 호출 연동합니다.
