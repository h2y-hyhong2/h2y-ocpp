# 💻 04. 구현 및 배포 정의서 (Implementation & Deployment)
> 가상 스레드 극대화와 로컬 맵 및 무인 인스톨러 배포

---

## 1. 개발 런타임 및 가상 스레드 스택
대용량 I/O 대기를 효율적으로 차단하고 CPU 효율성을 높이기 위한 백엔드 코어 스택 구현 요건입니다.

* **Java 25 가상 스레드(Virtual Threads) 연동:**
  * 기존 Spring의 OS 스레드 맵핑(1:1 Thread Pool) 방식 대신 JVM 레벨의 가상 스레드(Carrier Thread에 스케줄링)를 활성화.
  * `application.properties` 내 `spring.threads.virtual.enabled=true` 속성 설정.
  * DB 트랜잭션 대기, HTTP 외부 로밍 API 통신, 파일 입출력 등 대기 시간이 발생하는 I/O 블로킹 구간에서 물리 스레드가 블로킹되지 않고 즉시 다른 가상 스레드 작업을 수행하도록 하여 1대 VM에서도 동시 2,000개 이상의 스레드를 가볍게 제어.

## 2. 경량화 튜닝 및 리소스 제어 개발
Phase 1 단일 호스트 가상 머신(VM) 환경에서 미들웨어(Kafka, ClickHouse)가 과도하게 시스템 메모리를 점유해 OOM(Out of Memory)으로 사망하는 상황을 예방하기 위한 튜닝 스크립트 설계입니다.

* **Kafka Heap Size 및 Retention 제한:**
  * 환경변수 `export KAFKA_HEAP_OPTS="-Xms512M -Xmx512M"` 설정을 통해 JVM 힙 메모리를 최소화.
  * `server.properties` 내 `log.retention.hours=24` 및 `log.segment.bytes=536870912`(512MB) 설정으로 하루가 지난 백로그 세그먼트는 자동 삭제하여 디스크 고갈 위협 차단.
* **ClickHouse 메모리 쿼리 임계치 제어:**
  * `users.xml` 파일에 `<max_memory_usage>4000000000</max_memory_usage>` 설정 추가. 개별 분석 쿼리가 4GB 이상의 메모리를 무단 점유하는 것을 강제 차단하여 전체 OS 런타임 안정성 보장.

## 4. 무인 오프라인 인스톨러 패키징 개발
인프라 구성 요소(PostgreSQL, ClickHouse, Kafka, Java JVM, App Engine)가 설치되지 않은 호스트 환경에서도 터미널 명령 한 줄(원클릭)로 무인 설치가 완료되도록 빌드 파이프라인을 구축합니다.

* **Linux 환경 (Makeself 아카이브 빌드):**
  * Makeself 유틸리티를 활용하여 통합 Docker Compose 설치 스크립트, 컨테이너 백업 이미지(`ocpp-lite-images.tar`), `docker-compose.yml` 및 로컬 설정 셋업 파일들을 하나로 아카이빙.
  * 단일 바이너리 파일 `install.run` 실행 시, 호스트 내 Docker 설치 유무를 식별하고 Docker 엔진 설치 대행 및 백업 이미지 로드, 컨테이너 오케스트레이션 기동까지 무인 실행(`Silent Install`)을 완수하도록 `install.sh` 스크립트 작성.
* **Windows 환경 (Inno Setup 컴파일러):**
  * Inno Setup 컴파일러를 통해 설치용 UI 프레임과 리소스 파일을 단일 실행 파일 `setup.exe`로 패키징.
  * 설치 과정에서 백그라운드로 WSL2 및 Docker Desktop 설치 프로세스를 자동 셋업하고, 컨테이너 구동 배치 스크립트(`install.bat`)를 호출하여 실행 인프라를 일괄 완성.
